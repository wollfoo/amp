# Documentation Management

## Key Documents
- `@doc/project-roadmap.md` — Phases, milestones, progress
- `@doc/project-changelog.md` — Changes, features, fixes  
- `@doc/system-architecture.md` — Architecture decisions
- `@doc/code-standards.md` — Coding conventions

---

## Update Triggers
Update docs when:
- [ ] Phase status changes
- [ ] Major feature implemented
- [ ] Significant bug fixed
- [ ] Security patch applied
- [ ] Breaking change occurs

## Protocol
1. Read current status before updating
2. Maintain version consistency
3. Verify links and dates
4. Align with actual implementation
